# Security Policy

## Supported Versions

This project is a small static web application and does not currently maintain version-specific security support branches. Security fixes are applied on the `main` branch when accepted.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

If you discover a vulnerability:

1. Contact the repository owner privately through the contact details available on the GitHub profile.
2. Include a clear description of the issue, reproduction steps, potential impact, and any suggested remediation.
3. Allow reasonable time for investigation and remediation before any public disclosure.

For non-sensitive bugs or feature requests, use the public GitHub issue tracker.

## Security Notes

- Do not commit secrets, API keys, or private credentials to this repository.
- Review any third-party sharing or upload configuration, including Cloudinary presets, before using the project in production.
